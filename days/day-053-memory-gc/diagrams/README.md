# Day 053 — 内存管理与垃圾回收图解

## 1. Python 内存管理架构

```mermaid
graph TD
    A[Python 应用程序] --> B[内存分配器]
    B --> C[pymalloc]
    B --> D[操作系统]

    C --> E[对象分配]
    C --> F[内存池]

    E --> G[引用计数]
    E --> H[垃圾回收器]

    G --> I[自动回收]
    H --> J[标记-清除]
    H --> K[分代回收]
```

## 2. 引用计数机制

```mermaid
sequenceDiagram
    participant Var as 变量
    participant Obj as 对象

    Var->>Obj: 创建对象 (refcount=1)
    Var->>Obj: 赋值给另一个变量 (refcount=2)
    Var->>Obj: 放入列表 (refcount=3)

    Note over Obj: 对象仍然存活

    Var->>Obj: del 变量 (refcount=2)
    Var->>Obj: 从列表移除 (refcount=1)
    Var->>Obj: del 最后一个引用 (refcount=0)

    Note over Obj: 对象被立即回收
```

## 4. 分代回收原理

```mermaid
graph TD
    subgraph 第 0 代
        A0[新对象] --> B0[存活对象]
    end

    subgraph 第 1 代
        A1[从 0 代存活] --> B1[存活对象]
    end

    subgraph 第 2 代
        A2[从 1 代存活] --> B2[长期存活]
    end

    B0 -->|存活| A1
    B1 -->|存活| A2
```

## 7. 内存泄漏检测流程

```mermaid
graph TD
    A[开始检测] --> B[获取内存快照]
    B --> C[执行操作]
    C --> D[再次获取快照]
    D --> E[对比快照]
    E --> F{内存增长?}
    F -->|是| G[可能存在泄漏]
    F -->|否| H[内存正常]

    G --> I[使用 gc.get_referrers 追踪]
    I --> J[找出引用链]
    J --> K[修复泄漏]
```
