# Day 058 — 并发模型对比 图解

## 1. GIL 工作原理

```
┌──────────────────────────────────────────────────────┐
│                   CPython 解释器                      │
│                                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │              GIL (全局解释器锁)                  │  │
│  │        同一时刻只有一个线程持有 GIL              │  │
│  └─────────┬──────────┬──────────┬────────────────┘  │
│            │          │          │                    │
│     ┌──────▼───┐ ┌────▼────┐ ┌──▼──────┐            │
│     │ Thread 1 │ │Thread 2 │ │Thread 3 │            │
│     │ 持有 GIL │ │  等待   │ │  等待   │            │
│     └──────────┘ └─────────┘ └─────────┘            │
│            │                                          │
│     ┌──────▼──────────────────┐                      │
│     │   执行 Python 字节码     │                      │
│     │   (一次执行 N 条指令)    │                      │
│     └──────────┬──────────────┘                      │
│                │                                      │
│     ┌──────────▼──────────────┐                      │
│     │  I/O 操作 → 释放 GIL    │  或                   │
│     │  时间片到 → 释放 GIL     │                      │
│     └──────────┬──────────────┘                      │
│                │                                      │
│     ┌──────────▼──────────────┐                      │
│     │  其他线程获取 GIL 继续   │                      │
│     └─────────────────────────┘                      │
└──────────────────────────────────────────────────────┘
```

## 3. 选择决策流程图

```mermaid
flowchart TD
    A[需要并发处理] --> B{任务类型?}
    
    B -->|CPU 密集型| C{需要多核加速?}
    C -->|是| D[多进程]
    C -->|否| E[单线程/协程]
    
    B -->|I/O 密集型| F{并发量?}
    F -->|高并发 >100| G[协程]
    F -->|中并发 10-100| H[线程池]
    F -->|低并发 <10| I[单线程 + 异步]
    
    B -->|混合型| J[线程池 + 进程池]
    
    D --> K[绕过 GIL\n多核并行]
    G --> L[轻量级\n事件循环]
    H --> M[操作系统调度\nI/O 释放 GIL]
    J --> N[各取所长\n最佳平衡]
```
