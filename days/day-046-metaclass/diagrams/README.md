## 2. type() 的两种用法

```
type() 的两种用法
═══════════════════════════════════════════

用法一：查询类型
─────────────────
    type(42)  →  <class 'int'>
    type("hi") →  <class 'str'>

用法二：动态创建类
─────────────────
    MyClass = type('MyClass', (object,), {'x': 1})
    
    等价于:
    class MyClass:
        x = 1
```

## 4. 自定义元类 vs 装饰器 vs __init_subclass__

```
选择指南
═══════════════════════════════════════════════════

┌────────────────┬────────────┬────────────┬─────────────────┐
│   特性          │   元类     │  装饰器     │ __init_subclass__│
├────────────────┼────────────┼────────────┼─────────────────┤
│ 作用范围        │ 所有子类   │ 单个类      │ 子类            │
│ 复杂度         │ 高         │ 低          │ 中              │
│ 可读性         │ 差         │ 好          │ 好              │
│ 能否修改类名    │ 能         │ 不能        │ 不能            │
│ 能否阻止类创建  │ 能         │ 不能        │ 不能            │
└────────────────┴────────────┴────────────┴─────────────────┘

经验法则:
  能用装饰器 → 用装饰器
  能用 __init_subclass__ → 用 __init_subclass__
  都不行 → 才考虑元类
```

## 6. Meta 三方法对比

```
__new__ vs __init__ vs __call__
═══════════════════════════════════════════════════

class Meta(type):
    def __new__(mcs, name, bases, attrs):
        """创建类对象 — 返回 cls"""
        return super().__new__(mcs, name, bases, attrs)
    
    def __init__(cls, name, bases, attrs):
        """初始化已创建的类 — 无返回值"""
        super().__init__(name, bases, attrs)
    
    def __call__(cls, *args, **kwargs):
        """创建类的实例 — 返回 instance"""
        return super().__call__(*args, **kwargs)

调用时机:
  Meta.__new__   ← class Foo: 时调用（创建类）
  Meta.__init__  ← class Foo: 时调用（初始化类）
  Meta.__call__  ← Foo() 时调用（创建实例）
```
